# Project Title: NodeMesh-X1

## Subtitle: High-Performance RTL-to-GDSII Implementation of a NodeMesh IL Sensor Packet Router

### 1. Project Overview

* **Domain:** Digital IC Design / Physical Design
* **Core Objective:** To design, synthesize, and harden a custom hardware accelerator block for the **NodeMesh IL** architecture. The project focuses on converting a high-level architectural concept into a manufacturable **GDSII** blueprint using an automated EDA flow.
* **Key Design Goal:** Achieving timing closure at **250MHz** on the **SkyWater 130nm** Open Source PDK.

---

### 2. Technical Specifications

| Parameter | Specification |
| --- | --- |
| **Logic Function** | 4-Channel Arbitrated Packet Router (SPI-to-Mesh) |
| **Instruction Set** | Custom NodeMesh Protocol (16-bit Packet Headers) |
| **Target Frequency** | 250 MHz ($T_{clk} = 4ns$) |
| **Target Process** | SkyWater 130nm CMOS |
| **EDA Toolchain** | **OpenLane** (Yosys, OpenROAD, Magic, Netgen, KLayout) |
| **Verification** | Icarus Verilog + GTKWave (Behavioral & Post-Layout) |

---

### 3. Detailed Milestones & Deliverables

#### **Milestone 1: RTL Architecture & Functional Signoff**

* **Objective:** Translate the NodeMesh communication logic from C++/ESP32 logic into synthesizable Verilog.
* **Technical Focus:** Finite State Machine (FSM) design for packet arbitration.
* **Deliverable:** `nodemesh_router.v` and a comprehensive Testbench showing zero packet loss during 10,000 cycle stress tests.

#### **Milestone 2: Logic Synthesis & Initial Timing Analysis**

* **Objective:** Convert Verilog to a gate-level netlist using **Yosys**.
* **Technical Focus:** Analyzing the "Critical Path." Identifying which logic gates are creating the longest delays.
* **Deliverable:** Synthesis Report showing Estimated Gate Count and Initial Slack.

#### **Milestone 3: Floorplanning & Power Distribution (PD Focus)**

* **Objective:** Define the physical boundaries of the chip and the "Power Well."
* **Technical Focus:** Placing I/O pins strategically for the Mesh interconnect to minimize wire length. Designing the **Power Distribution Network (PDN)** to prevent voltage drops.
* **Deliverable:** Floorplan DEF file showing macro placement and metal power grids.

#### **Milestone 4: Placement & Clock Tree Synthesis (CTS)**

* **Objective:** Physically place the millions of standard cells and build the "Heartbeat" (Clock) of the chip.
* **Technical Focus:** Minimizing **Clock Skew** (ensuring the clock signal hits every gate at the exact same time).
* **Deliverable:** Post-CTS Timing Report with focus on **Hold Time** violations.

#### **Milestone 5: Routing & Timing Closure**

* **Objective:** Use the global and detail routers to connect all gates with metal layers (Met1 - Met5).
* **Technical Focus:** Fixing **Antenna Violations** and **Setup Slack**. This is the iterative "Loop" of Physical Design.
* **Deliverable:** Final Routing DEF and Static Timing Analysis (STA) Signoff.

#### **Milestone 6: Physical Verification (Layout Focus)**

* **Objective:** Ensure the chip can actually be manufactured without errors.
* **Technical Focus:** **DRC** (Design Rule Check) to ensure wires aren't too close, and **LVS** (Layout vs. Schematic) to ensure the physical shapes match the original logic.
* **Deliverable:** DRC/LVS Clean Report and the final **GDSII** file.

### 4. Skills & Competencies Demonstrated

* **RTL Design:** Mastery of synthesizable Verilog and hardware handshaking protocols.
* **STA Mastery:** Deep understanding of $T_{setup}$ and $T_{hold}$ constraints.
* **EDA Automation:** Ability to configure and debug automated flows using **Tcl/Python**.
* **Spatial Reasoning:** Optimization of floorplans to reduce congestion and "Hotspots."
* **Foundry Awareness:** Working within the strict geometric constraints of the SkyWater 130nm PDK.
